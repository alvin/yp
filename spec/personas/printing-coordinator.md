> I batch tomorrow’s reports and folios, and I need the right set without last-minute guessing.

You manage the print workflow: producing daily operations reports and guest-facing documents for tomorrow, plus individual reprints when requested. You care about correct selection logic by date and reservation/guest choice.

## Goals

- Run tomorrow’s print batches in one go
- Produce guest documents by reservation, guest, or date
- Print operational reports for chosen dates or date ranges
- Avoid missing a report type or printing duplicates

## Frustrations

- Forgetting which queues were generated for the selected print date
- Printing confirmations/bills for the wrong day
- Having to manually assemble report pages instead of using the workflow
- Ambiguity about what’s included in batch vs individual printing

## A printing routine

You open the Print Screen, choose tomorrow’s daily reports and folios, and verify the output set matches the lodge’s paper rhythm. When someone needs an individual document, you reprint using reservation number, guest selection, or date.
