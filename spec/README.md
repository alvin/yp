# Yellow Point — Product Spec

The living specification for **Yellow Point**, kept in sync with
[UserForge](https://userforge.com). Written to be built from — by engineers and
coding agents — while stakeholders track progress in the UserForge dashboard.

_6 personas · 104 features · 24 wireframes._

## What's here

| Folder | What it is |
| --- | --- |
| `personas/` | **Who** we're building for — one Markdown profile each |
| `features/` | **What & why** — user stories with acceptance criteria, as Gherkin `.feature` |
| `wireframes/` | **Where** — an SVG sketch of each screen / flow |

## Building from this spec

1. Read `features/*.feature` — each is one user story; the `Then / And …` steps
   are its acceptance criteria, and `@persona:` links the person it serves.
2. Open the matching `wireframes/*.svg` for the intended UI.
3. As you build, update each feature's status tag — it syncs back to every
   stakeholder's board on the next sync:

```gherkin
@persona:returning-shopper @status:in-progress @priority:4
Feature: reorder a previous purchase
```

Statuses: `@status:backlog`, `planned`, `in-progress`, `done`, `validated`.

## Good to know

- Keep the `# id:` lines and frontmatter `id:` fields — that's how an edit syncs
  back to the right item instead of duplicating.
- A new `.feature`, `.md` or `.svg` without an id imports as a brand-new item.
- Syncing in only creates and updates — it never deletes.
