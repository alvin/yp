Screen Notes — The ‘Print Screen’

This should hopefully be a fairly straightforward page with numerous buttons to select either the 8.5 x 11” standard paper reports or the smaller ‘A2’ folios for confirmations and check ins, etc.

On most days this would be a ‘batch job’ with one button sending the command to print all the reports, automatically selected for ‘tomorrow’. However, when the individual buttons are selected, a date and/or name can be input to produce an individual copy. This is useful when a guest needs a new confirmation or we have made some changes to their bill upon check out and they need a copy of the correct one, but we don’t want to run a whole batch.

This is a detail we’ll work out, but maybe when the individual ‘confirmation’, ‘cancellation’ or ‘Bill’ button is selected the user may be prompted with a choice of either ‘all confirmations for today’ or ‘select individual confirmation’, or something like that.
