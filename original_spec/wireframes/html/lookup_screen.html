Screen Notes — The ‘Lookup’ Screen

This will be the ‘home screen’, used to search for ‘Transaction’ screens either by entering the guests last name, (which will produce a steadily reducing list of choices as more characters are entered), their existing reservation number or a date (selectable by one of those little pop-up calendars). A useful feature would be the ability to enter a partial string and get a name match. Eg: if we have a couple with two last names like Winston McAdam we could enter ‘Adam’ and it would return any names with that string.

The user can also access a blank Transaction Screen, (with ‘Guest Info’ blank as well, in case it’s a new guest) screen to make a new reservation and go to the Print Screen and Daily Cash Activity Report.

The lookup screen also supports all-fields search for cases where staff only have a phone number or partial address, and date-range access in addition to single-day date search. All-fields access gating is tracked in lookup_screen.questions.
