Screen Notes — ‘Name Search’ return screen

This screen is returned when a name is searched. It lists the guest information on the left and a list of all their reservations on the right. The reservation list is broken into three sections, ‘Past Reservations’ ‘In-House’ and ‘Future Reservations’. (We are hoping this is possible instead of our present set-up where any reservation older than a year is ‘archived’ and must be searched separately).

The user will select the date that they want for that particular guest. They may be looking up payment information on a past stay, adding a charge or name to a current reservation, putting a deposit or payment on a future one, or simply providing information to the guest.

When a date is selected, the Transaction Screen is returned.

There is a button called ‘guest notes’ that brings up a text screen that is only viewable on screen and is particular to the guest themselves and is not detail about a single reservation. This information should not print to any report because it is usually sensitive, but required by the office. Examples may include touchy things like ‘credit card is usually declined’, ‘Hygiene / continence issues’, ‘smoked in room during visit’, ‘Idiot’, etc.

In the ‘Guest Info’ section, we’ll need to talk about how we handle reservations with multiple names, ie: couples, people sharing a room, etc. Maybe there could be ‘stacked’ guest info screens when a box was checked or something.
